# Intent

Quiz runs the two scored study modes — Practice (immediate feedback, unlimited retries) and Test (no feedback until the end, freely changeable answers). It exists to give the student a real quiz experience with free navigation (skip around, come back later) while still scoring fairly: Practice scores the first attempt at each question so retrying to learn never inflates the score, and Test scores whatever the FINAL chosen answer was when the session ends, since nothing is revealed mid-test to make an earlier answer special. A question never answered by the time the session ends is scored wrong, not omitted.

# Requirements

- R1 [verify: ui] The quiz presents one question at a time with its text (KaTeX-rendered), an optional graph above the text when the question has a `graph`, and exactly four answer buttons labeled A–D.
- R2 [verify: ui] A progress indicator shows `{current} of {total}` and a progress bar fills proportionally as the student advances. [caution: the bar fills by `(current-1)/total` — i.e. it reflects questions completed before the current one, so it reads 0% on Q1. Preserve this "completed" semantic rather than switching to `current/total`.]
- R3 [verify: unit] [caution: in Practice, `firstAttemptCorrect` is set on the very first submission for a question and is NEVER overwritten by later retries — that's still the core anti-gaming invariant. In Test there is no "first attempt": no feedback is shown, so changing an answer via Back isn't a retry, it's just picking again — `firstAttemptCorrect` there reflects the FINAL chosen answer as of session end.] At session end, every question in the session (not just answered ones) yields exactly one stat entry: `{ id, firstAttemptCorrect, chosenIndex, correctIndex, timeSpent }`. An unanswered question records `chosenIndex: -1` and `firstAttemptCorrect: false`.
- R4 [verify: ui] Practice mode: selecting an answer immediately shows right/wrong feedback. A correct pick turns green and disables all answers (locked, even after navigating away and back); a wrong pick turns that answer red, leaves the other answers enabled so the student can keep guessing, and shows an "Incorrect — try again" message. [caution: on each retry, clear the previous wrong highlight before marking the new pick; retries after the first must NOT change the recorded first-attempt stat. Revisiting an already-answered question via Back/Next restores its saved state instead of resetting it.]
- R5 [verify: ui] Practice mode: a "Copy explanation prompt" button lives in the middle of the Back/Next row (same slot Test uses for its timer — see R14) and is ALWAYS visible there, not gated on having answered. It copies the AI prompt to the clipboard with brief "Copied!" feedback. Before the question is answered, the prompt just asks to explain the correct answer (no chosen index); once answered, it reflects the MOST-RECENTLY chosen index for that question. No copy button in Test mode (Stats' breakdown covers it there) — this is a deliberate mode difference, not an oversight. The "Correct!"/"Incorrect" feedback message stays in its own row above the nav row, still gated on `answered`.
- R6 [verify: ui] Test mode: selecting an answer highlights it as the current selection (no green/red, no correctness feedback). The selection can be changed at any time, including after navigating away via Back/Next and returning — there's no lock and no retry-vs-first-attempt distinction while the session is live.
- R7 [verify: unit] Test mode: `timeSpent` per question accumulates in whole seconds across every visit to that question (Back navigation doesn't reset it — time from each visit adds to the running total), finalized when the session ends. [caution: `timeSpent` is populated only in Test mode; in Practice it is null.]
- R8 [verify: ui] A Next control advances to the next question; its label reads "Next →" for all but the last question, and "See Results" on the last question. Both Back and Next are ALWAYS visible and enabled (Back disabled only on the first question) — neither requires answering the current question first, so the student can skip through the whole deck unanswered if they want. Advancing past the last question ends the session and shows Stats.
- R9 [verify: unit] The quiz owns a session timer: total wall-clock duration in whole seconds from quiz start to the Stats hand-off is computed and passed into the session record.
- R10 [verify: ui] Keyboard: keys `1`–`4` select the corresponding answer (only if that button is enabled); `ArrowLeft` moves to the previous question (no-op on the first question); `Enter` or `ArrowRight` moves to the next question, or finishes the session on the last question — none of this requires answering first. [caution: number keys must respect the disabled state of answer buttons — e.g. in Practice after a correct answer all are disabled and `1`–`4` should do nothing.]
- R11 [verify: ui] An abandon/quit control (✕) exits the quiz WITHOUT producing a Stats record. [caution: it returns to Mode Select (the Practice/Test/Review picker for the current file), NOT all the way to Home — Quiz is only ever entered from Mode Select, so that's always the correct place to land. Do not revert this to Home.]
- R12 [verify: unit] At session end, per-question answer records covering every question in the session (synthesizing wrong/unanswered entries for anything skipped), total duration, and mode are handed to Stats to build the session record, and the app navigates to the Stats screen. [caution: scoring/aggregation lives in Stats, not Quiz — Quiz emits raw per-question records only, keeping the two independently evolvable.]
- R13 [verify: ui] Navigating to a question (first visit or a revisit via Back/Next/arrow keys) restores that question's saved answer state — chosen index, correctness highlight/lock in Practice, pending selection in Test — rather than resetting it. Only the "Copy explanation prompt" button's label resets to default on navigation.
- R14 [verify: ui] The Back/Next row has one explicit-grid-column middle slot, used differently per mode: Test shows a live session clock, starting at 0 and ticking every second from when the quiz started (not per-question — Back/Next navigation doesn't reset or pause it), using the same `formatDuration` format as Stats' "Session time" line so the number matches what's shown on the summary afterward. Practice shows the "Copy explanation prompt" button there instead (R5). Never both — they're mutually exclusive by mode.

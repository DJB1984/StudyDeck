# Intent

Quiz runs the two scored study modes — Practice (immediate feedback, unlimited retries) and Test (no feedback until the end, one shot per question). It exists to give the student a real quiz experience while scoring only the first attempt at each question, so that retrying in Practice to learn a concept never inflates or corrupts their measured performance.

# Requirements

- R1 [verify: ui] The quiz presents one question at a time with its text (KaTeX-rendered), an optional graph above the text when the question has a `graph`, and exactly four answer buttons labeled A–D.
- R2 [verify: ui] A progress indicator shows `{current} of {total}` and a progress bar fills proportionally as the student advances. [caution: the legacy bar fills by `(current-1)/total` — i.e. it reflects questions completed before the current one, so it reads 0% on Q1. Preserve this "completed" semantic rather than switching to `current/total`.]
- R3 [verify: unit] [caution: FIRST-ATTEMPT-ONLY scoring is the core invariant of this whole feature. `firstAttemptCorrect` is set on the very first submission for a question and is NEVER overwritten by later retries. Guard submission so only the first call per question records a result; subsequent clicks on that question are ignored for stats.] Each answered question records exactly one stat entry: `{ id, firstAttemptCorrect, chosenIndex, correctIndex, timeSpent }`.
- R4 [verify: ui] Practice mode: selecting an answer immediately shows right/wrong feedback. A correct pick turns green and disables all answers; a wrong pick turns that answer red, leaves the other answers enabled so the student can keep guessing, and shows an "Incorrect — try again" message. [caution: on each retry, clear the previous wrong highlight before marking the new pick, so the student sees which one they just chose; retries after the first must NOT change the recorded first-attempt stat.]
- R5 [verify: ui] Practice mode: after the first selection, a "Copy explanation prompt" button appears that copies the AI prompt (built from the question and the chosen answer) to the clipboard with brief "Copied!" feedback. The prompt is rebuilt on every pick and reflects the MOST-RECENTLY chosen index (i.e. on a retry the prompt describes the answer just re-picked), matching legacy behavior. [caution: this is deliberately the latest pick, NOT the first-attempt index — the legacy app reassigns the copy button's handler on each practice click (studydeck.html handlePracticeClick). Do not "correct" this to the first-attempt index; the Stats breakdown copy button separately uses the first-attempt `chosenIndex`, so both surfaces are already covered.]
- R6 [verify: ui] Test mode: selecting an answer highlights it as the pending selection (no green/red, no correctness feedback) and lets the student change selection freely until they advance. No retry semantics and no copy button in Test mode.
- R7 [verify: unit] Test mode: the answer is recorded on advancing to the next question (submitting the pending selection), and `timeSpent` for that question is captured in whole seconds from when the question rendered to submission. [caution: `timeSpent` is populated only in Test mode; in Practice it is null.]
- R8 [verify: ui] A Next control advances to the next question; its label reads "Next →" for all but the last question, and "See Results" on the last question. Advancing past the last question ends the session and shows Stats.
- R9 [verify: unit] The quiz owns a session timer: total wall-clock duration in whole seconds from quiz start to the Stats hand-off is computed and passed into the session record.
- R10 [verify: ui] Keyboard: keys `1`–`4` select the corresponding answer (only if that button is enabled); `Enter` or `ArrowRight` triggers Next when the Next control is visible. [caution: number keys must respect the disabled state of answer buttons — e.g. in Practice after a correct answer all are disabled and `1`–`4` should do nothing.]
- R11 [verify: ui] An abandon/quit control exits the quiz directly to Home without producing a Stats record.
- R12 [verify: unit] At session end, the answer records, total duration, and mode are handed to Stats to build the session record, and the app navigates to the Stats screen. [caution: scoring/aggregation lives in Stats, not Quiz — Quiz emits raw per-question records only, keeping the two independently evolvable.]
- R13 [verify: ui] On each question render the pending Test selection is reset, feedback is hidden, answer buttons are re-enabled and cleared of highlight, and the copy button is hidden.

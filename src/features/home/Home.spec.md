# Intent

Home is the landing screen and the entry point for every study session. It lets a student re-open any previously loaded deck instantly, add new decks by drag-drop, file picker, or pasted text, remove decks they no longer want, and — because StudyDeck ships no built-in AI — get a study set made by any external AI. The screen must be understandable at a glance by a non-technical first-time user: the tagline says what the app does, and the get-started path is short, plain-English, and free of developer jargon.

# Requirements

- R1 [verify: ui] Home renders a grid of file cards, one per entry in `Storage.getHistory()`, each showing the deck `title` and a metadata line `{count} questions · {lastOpened}`.
- R2 [verify: ui] Cards appear newest-first, matching the order Storage returns (Storage prepends new/updated entries). [caution: ordering comes from Storage's saveFile prepend behavior — Home must render history in the array order it receives, not re-sort it.]
- R3 [verify: ui] Each card has a delete control. Clicking it removes that deck via `Storage.deleteFile(title)` and re-renders the grid, WITHOUT navigating away from Home. [caution: the delete click must not also trigger the card's open action — in the legacy app this is `stopPropagation`; in React, stop the event from bubbling to the card's click handler.]
- R4 [verify: ui] Clicking a card (anywhere but the delete control) updates that entry's `lastOpened` to today, re-saves it via `Storage.saveFile`, refreshes the grid, and navigates to the deck's matching screen: quiz decks → Mode Select; flashcard decks → the Flashcard screen directly. [caution: the type-based destination is decided by App's `onOpenDeck` handler, not inside Home — Home just reports "this deck was opened" and must not grow its own routing branch.]
- R5 [verify: ui] A drop zone (the add-deck surface) accepts a dragged file; on drop, the file is routed into the load flow. Dragging over the zone applies a visible highlight state that clears on drag-leave or drop.
- R6 [verify: ui] The drop handler rejects any file whose name does not end in `.json` (case-insensitive) with the error `Please drop a .json file.` and does not attempt to parse it.
- R7 [verify: ui] A "Load file" button opens a native file picker restricted to `.json`; choosing a file routes it into the same load flow as drop. [caution: after a pick, reset the input value so re-selecting the same file fires the change event again.]
- R8 [verify: ui] The load flow reads the file text, `JSON.parse`s it, and on parse failure shows the error `Invalid JSON: {message}` without adding anything to history.
- R9 [verify: ui] On successful parse, the file is validated via DeckValidation; if any errors are returned, they are shown joined by newlines and the file is NOT saved. Only a clean validation proceeds.
- R10 [verify: ui] On a valid deck, a history entry `{ name, title, count: questions.length, lastOpened: today, data: parsedDeck }` is saved via `Storage.saveFile`, the deck becomes the current file, the grid re-renders, and the app navigates to the deck's matching screen (quiz → Mode Select; flashcard → Flashcard directly, same App-level routing as R4).
- R11 [verify: ui] When history is empty, the get-started card (R12) renders as the hero — above the add-deck surface — and there is NO separate empty-state message. The card itself is the empty state. [caution: do not reintroduce a "no study sets loaded yet" sentence alongside the card; the redesign deliberately removed the duplicate copy — the card alone carries the first-run guidance.]
- R12 [verify: ui] The get-started card is headed "Make a study set with any AI" (or near-identical plain-English copy) and contains exactly 3 plain-English steps with ONE primary button labeled "Copy Prompt": (1) copy the prompt; (2) paste it into any AI, phrased purely as a handoff (e.g. "Paste it into ChatGPT — or any AI — and it'll take it from there") — step 2 must NOT mention notes or instruct the student to attach anything, because the pasted prompt itself asks for their notes and turns the AI into a helper agent that either produces the set immediately or tailors it through conversation; (3) paste the FINISHED study set back here (e.g. "Paste the finished study set back here") — "finished/final", not "the AI's reply", since tailoring may happen in the chat before delivery. The words "format spec" and "JSON" must NOT appear anywhere in the card's user-facing text. [caution: only the button LABEL is plain-English — clicking it opens a full-screen Quick/Guided choice modal rather than copying immediately (see R12a). Do not swap in a shortened "friendly" prompt payload for either variant. And do not "helpfully" re-add "along with your notes" to step 2 — asking for notes deliberately lives inside the pasted prompt, not on the card.]
- R12a [verify: ui] Clicking "Copy Prompt" (in the card or the R22 header corner — same `CopyPromptButton` component, same behavior in both places) opens a full-screen modal (`CopyPromptModal`, styled like `LoginModal`'s overlay — dimmed backdrop, centered card, `×` close button, click-outside-to-close) with two choice cards (styled like ModeSelect's `.mode-card`): **Quick** ("Paste your notes and go — fast, no questions asked.") and **Guided** ("A few quick questions first, so the set is tailored to what you actually need."). Picking a card copies that variant's composed prompt (`QUICK_PROMPT_MD` or `GUIDED_PROMPT_MD` from `src/lib/formatSpec.ts` — each is a per-variant intro file plus the shared `studydeck-format-spec.md` schema contract, `?raw` build-time imports so nothing drifts and it works offline), closes the modal, and flips the button label to "Copied!" via the existing `copyWithFeedback` (unchanged), same as before the modal existed. [caution: this replaced an earlier small anchored-popover version (2026-07-21) — Davis: "it should pop up a page similar to the Login page where you have to choose on the screen, rather than having an ugly dropdown." Don't revert to a popover/dropdown.]
- R13 [verify: ui] Errors surface through the shared toast (behavior owned by `src/components/Toast.spec.md`: manually dismissible, auto-dismissing, multi-line). Home's obligation is only to route every load-flow error through `showError` — never `alert`, inline text, or console-only.
- R14 [verify: ui] The subtitle under the "StudyDeck" heading reads "Use ⟨AI⟩ to turn your notes into ⟨output⟩" (final phrasing may vary slightly, but it must describe what the app does) with TWO rotating word slots: the AI slot cycles through a call-site list of well-known AI names (per R17; initially ChatGPT, Claude, Gemini, Grok — since extended, e.g. Copilot, Perplexity); the output slot cycles through flashcards, quizzes, tests (these mirror the app's actual modes and are not freely extensible). - R18 [verify: ui] The add-deck surface carries a "Paste study set" control alongside drag-drop and "Load file". Clear empties the textarea but KEEPS the box open; Cancel discards the text AND collapses the box — these are distinct actions and must stay so. Clear is disabled when the textarea is empty (the same non-blank condition as Add). Neither Clear nor Cancel has any other side effect. [caution: don't merge Clear into Cancel or promote either to primary styling — Add Study Set must remain the lone primary action in the box.]
